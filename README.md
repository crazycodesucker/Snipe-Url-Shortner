## рҹӣЎпёҸ Security Audit

> *"Working and production-ready are not the same thing."*

Before pushing to production, I put on the attacker hat and went through the entire app. Here's the full audit report:

<br/>

| рҹ”Қ | Vulnerability | Severity | Status |
|:--:|--------------|:--------:|:------:|
| 1 | **No rate limiting** вҖ” `/shorten` wide open for mass slug generation | рҹ”ҙ High | вң… Fixed |
| 2 | **Slug enumeration** вҖ” 6-char keyspace (~56B) is bruteforceable | рҹҹЎ Medium | вң… Fixed |
| 3 | **SSRF via URL input** вҖ” `file://`, `javascript://` accepted | рҹ”ҙ Critical | вң… Fixed |
| 4 | **AWS metadata exposure** вҖ” `169.254.169.254` not blocked | рҹ”ҙ Critical | вң… Fixed |
| 5 | **Path traversal on slugs** вҖ” special chars not rejected | рҹҹЎ Medium | вң… Fixed |
| 6 | **Click counter race condition** вҖ” read-modify-write under load | рҹҹў Low | вң… Fixed |
| 7 | **DB session leaks** вҖ” `db.close()` skipped on exceptions | рҹҹЎ Medium | вң… Fixed |
| 8 | **Zero security headers** вҖ” no CSP, server fingerprint exposed | рҹҹЎ Medium | вң… Fixed |
| 9 | **No request size limit** вҖ” oversized bodies hit the app | рҹҹў Low | вң… Fixed |

<br/>

### рҹ”’ What the fixes look like

**Rate Limiting** вҖ” Sliding-window counter per IP, per endpoint:
```
POST /shorten   вҶ’  10 req / min / IP
GET  /{slug}    вҶ’  60 req / min / IP
GET  /stats     вҶ’  30 req / min / IP
```

**SSRF Protection** вҖ” Multi-layer URL validation:
```python
BLOCKED_SCHEMES  вҶ’  file, javascript, data, ftp, vbscript, blob
BLOCKED_HOSTS    вҶ’  localhost, 169.254.169.254, metadata.google.internal
PRIVATE_RANGES   вҶ’  127.x, 10.x, 172.16-31.x, 192.168.x, 169.254.x, ::1
```

**Security Headers** вҖ” Applied to every single response:
```
X-Content-Type-Options    вҶ’  nosniff
X-Frame-Options           вҶ’  DENY
X-XSS-Protection          вҶ’  1; mode=block
Content-Security-Policy   вҶ’  default-src 'none'; frame-ancestors 'none'
Referrer-Policy           вҶ’  strict-origin-when-cross-origin
Server header             вҶ’  stripped (no fingerprinting)
```

**Slug Hardening:**
```
Before  вҶ’  6 chars  вҶ’  ~56,000,000,000 combinations
After   вҶ’  8 chars  вҶ’  ~218,000,000,000,000 combinations
Regex   вҶ’  ^[A-Za-z0-9]{6,16}$ enforced before DB is ever touched
```

## вҡҷпёҸ API Reference

### `POST /shorten`
> Shorten a URL and get a slug back.

```bash
curl -X POST http://localhost:8000/shorten \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/some/very/long/url"}'
```

```json
{
  "short_code": "aB3xKp9Z",
  "url": "https://example.com/some/very/long/url",
  "clicks": 0
}
```

---

### `GET /{slug}`
> Redirects to original URL. Click count incremented atomically.

```bash
curl -L http://localhost:8000/aB3xKp9Z
# вҶ’ 301 Redirect to original URL
```

---

### `GET /stats/{slug}`
> Fetch analytics for a slug.

```bash
curl http://localhost:8000/stats/aB3xKp9Z
```

```json
{
  "short_code": "aB3xKp9Z",
  "url": "https://example.com/some/very/long/url",
  "clicks": 42
}
```

---

### `GET /health`
> DB-connected health check.

```bash
curl http://localhost:8000/health
# вҶ’ {"status": "ok"}
```
